Hypertext Transfer Protocol

Versions

Throughout the over 20 year lifespan of HTTP, the protocol has gone throught many upgrades and itterations. The different versions of HTTP have focused on different upgrades; like security, efficinecy, and features.

HTTP 0.9
  • Original version of protocol.
  • Very basic.
  • Only had the GET method included.
HTTP 1.0
  • Introduced HTTP Headers - Allowing metadata to be transfered through requests and responses.
  • Introduced Status Codes - These status codes allowed meta information relating to request responses to be sent to clients.
  • Introduced Other File Sending - This allowed HTTP to be used to send more that just HTTP files.
HTTP 1.1
  • Introduced Pipelining - This fixed the bottleneck of having to wait for a response before sending another request. Fixing the pipelinging issues drastically improved preformance.
  • Introduced Reusable Connections - Unlike previous versions of HTTP, this version doesn't need to create multiple TCP connections for each request, but can use one for multiple.
  • Introduced 'host' Header - Allowing different webpages to be hosted on a machine with a single IP address.
HTTPS
  • Encrypted - Utilizing a encryption protocol called SSL alongside HTTP, finally allowed HTTP to be secure
  • SSL and TLS - SSL would continuously get security and efficiency updates, and eventually be renamed to TLS
  • Both HTTP and TLS are application layer protocols, and are used in conjuction. These two protocols would later be integrated to become more efficient.
HTTP 2
  • Binary Protocol - This changed transmitted data from text to direct binary data. Due to this change, many preformance impprovements could be made, like making information sent more condensed, allowing for compression, etc.
  • Introduced Multiplexing - Similar to piplineing but on a much lower level, it allows multiple parts of requests and responses to be sent and received out of order.
  • Other Preformance Optimizations
HTTP 3
  • New Transport Protocol - Unlike previous versions of HTTP, this version trades the oldschool TCP/IP suite for a newer QUIC transport protocol.
  • UDP Based - The new QUIC protocol is UDP based. Unlike TCP/IP, UDP based protocols are connectionless, allowing for significantly less overhead and speed.
  • Encrypted - QUIC has built-in encryption technology, finnally giving a version of HTTP native encryption. Unlike previous versions, the TLS encryption handshakes happen within the transport layer protocol, instead of as a seperate interaction on top of it.

NOTE While This section covers major improvements and versions, there have been smaller improvements made between these versions that I did not include.
